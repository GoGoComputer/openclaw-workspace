## 1. 일주일이 걸리던 설치 — 양파 까기

먼저 옛날 얘기를 잠깐 하겠습니다. 한 십 년쯤 전에는 자동화 도구 하나를 내 컴퓨터에 깔려면 진짜로 일주일이 걸렸어요. 일주일이요. 농담 아닙니다. 설치 안내 문서가 한 사십 페이지짜리예요. 첫 페이지부터 *"먼저 파이썬 3.7 이상이 설치되어 있어야 합니다"* 하고 시작합니다. 파이썬을 깔러 갑니다. 파이썬을 까는 데 또 뭐가 필요해요. 컴파일러가 필요하대요. 컴파일러를 깔려면 또 다른 게 필요하고요.

이게 끝없이 이어집니다. 그때 썼던 표현이 **양파 같다**. 까도 까도 안에 또 다른 게 나와요. 그 양파를 다 까고 나면 이번엔 환경 변수라는 걸 설정해야 한대요. 그게 또 뭐예요. 그러면 또 검색합니다. 그렇게 일주일이 지나고 나면 어떤 일이 벌어지냐. 처음에 왜 이걸 깔려고 했는지를 까먹어요. 진짜예요. 분명히 어떤 업무 하나 자동화하려고 시작한 건데, 일주일을 헤매다 보면 *"내가 지금 뭐 하고 있지"* 이 상태가 됩니다.

그리고 그 시점에서 한 절반 정도는 포기해요. *"아, 됐다. 그냥 손으로 하자."* **이게 자동화가 우리 일상으로 안 들어왔던 가장 큰 이유 중 하나입니다.** 시작하기가 너무 어려워서.

## 2. 누군가가 깎아둔 양파 — `./openclaw install`

시간이 지나면서 이 양파 까는 일을 누가 좀 대신해 주면 안 되나, 라는 생각을 사람들이 하기 시작합니다. 그래서 나온 게 우리가 만질 한 줄짜리 명령어예요.

```text
./openclaw install
```

점, 슬래시, 오픈클로, 스페이스, 인스톨. 끝이에요. 이 한 줄을 터미널에 입력하고 엔터를 치면, 십 년 전에 일주일 동안 했던 그 모든 양파 까기를 이 친구가 알아서 합니다. 한 십 분에서 이십 분 정도 걸려요. 그동안 커피 한 잔 드시면 됩니다.

이게 왜 대단한 일인지, 비개발자분들은 처음엔 잘 와닿지 않으실 거예요. *"원래 그런 거 아니에요?"* 라고 생각하시기 쉬워요. **아니에요. 원래 그런 거 아니었습니다.** 누군가가 그 양파 까는 작업을 한 줄짜리 스크립트 안에 차곡차곡 정리해 둔 거예요. 그 스크립트의 전문(全文) 은 [부록 A1 — `cmd/install.sh`](../appendix-a-scripts/A1-cmd-install.md) 에서 직접 보실 수 있습니다.

## 3. 한 줄 안을 들여다보기 — 여덟 단계 개관

이 한 줄이 안에서 뭘 하는지 한번 같이 들여다봅시다. 코드를 보자는 게 아니라 *"어떤 일들이 순서대로 일어나는지"* 그림을 그려보는 거예요. 머리 속에 이 그림이 한 번 그려지면, 나중에 어디서 멈춰도 *"아 지금 어디쯤이구나"* 감이 옵니다.

| 단계 | 하는 일 | 시간 |
|---|---|---|
| 1 | 자기소개 + 맥 신분증 확인 | 1분 |
| 2 | 맥 기본 도구함 점검 (Xcode CLT + Homebrew) | 5~10분 |
| 3 | 도커 데스크탑 설치 + 데몬 띄우기 | 5~10분 |
| 4 | 올라마 설치 (도서관 건물만) | 1~2분 |
| 5 | 오픈클로 조립도 다운로드 | 1분 |
| 6 | `.env` 비밀번호장 만들기 | 즉시 |
| 7 | 컨테이너 방 만들고 깨우기 | 5~10분 |
| 8 | 최종 건강검진 | 1분 |

## 4. 1단계 · 손님맞이 준비 (맥 신분증 확인)

첫 번째로 하는 일은 손님맞이 준비입니다. 인스톨러가 자기소개를 해요. 그리고 한 가지 중요한 걸 물어봅니다. *"지금 이 맥북, 인텔이에요 애플 실리콘이에요?"*

왜 묻냐면, 인텔 맥과 M1·M2·M3·M4 같은 애플 실리콘 맥은 안에서 돌아가는 부품이 좀 달라요. 그래서 깔아야 할 도커 버전도 다르고, 올라마가 쓸 수 있는 AI 모델 종류도 좀 달라집니다. 가장 먼저 우리 맥의 신분증을 확인하는 거예요.

> 💻 **직접 해보기** — 1단계 수동 버전: 맥 신분증 확인

```bash
uname -m            # arm64 = Apple Silicon / x86_64 = Intel
sw_vers             # macOS 버전
```

> 🔍 구현 위치: [`lib/detect.sh`](../appendix-a-scripts/A6-lib-rest.md#a6-1-libdetectsh--os아키텍처-감지)

## 5. 2단계 · 도구함 점검

두 번째 단계는 도구함 점검입니다. 맥에 원래부터 들어 있는 도구 중에 우리가 꼭 필요한 두 가지가 있어요.

- **Xcode Command Line Tools** — 맥이 새 프로그램을 깔 때 쓰는 기본 연장 세트. 망치, 드라이버, 펜치 같은 것.
- **Homebrew** — 맥용 앱 스토어인데 터미널에서 쓰는 버전. 마우스로 클릭하는 게 아니라 글자로 *"이거 깔아줘"* 하면 알아서 깔아주는 친구.

인스톨러가 둘이 있는지 확인하고, 없으면 깔아줍니다. 있으면 *"오, 이미 있네, 패스"* 하고 넘어가요. 이게 좀 중요한 포인트예요. **이미 있으면 또 깔지 않습니다.** 어려운 말로 **멱등성**이라고 하는데, [7장](../part4-deep/07-internals.md) 에서 본격적으로 다룰 겁니다. 지금은 *"있으면 안 깔고 넘어간다, 그래서 두 번 세 번 돌려도 안전하다"* 정도로 기억하시면 돼요.

> 💻 **직접 해보기** — 2단계 수동 버전: Xcode CLT 설치 + git 확인

```bash
xcode-select --install      # 다이얼로그 → "설치" 클릭 → 5~10분
# 이미 깔려 있으면: "command line tools are already installed" (정상)
git --version               # git version 2.x = OK
```

## 6. 3단계 · 도커 데스크탑 설치

세 번째가 진짜 큰 친구입니다. 도커 데스크탑이에요. 2장에서 도커를 호텔 객실로 비유했죠. 그 도커를 이 단계에서 깝니다.

도커 데스크탑은 용량이 좀 커요. 한 700MB ~ 1GB 정도. 인터넷 속도에 따라서 시간이 좀 걸립니다. 다 받으면 깔고, 깔고 나면 도커가 백그라운드에서 돌아가도록 시작 버튼을 눌러줘요. 우리가 아무것도 안 해도 됩니다.

설치가 끝나면 **도크에 도커 데스크탑 아이콘이 새로 생깁니다.** 고래 모양 아이콘. 그 친구가 이제부터 우리 맥 한쪽에 살면서 컨테이너들을 관리해 줄 거예요. 도크에 있는 그 고래 보이시면 마음속으로 한 번 인사해 주세요. *"안녕, 잘 부탁해."*

> 💻 **직접 해보기** — 3단계 수동 버전: Docker Desktop 실행 + 데몬 확인

```bash
# (사전) https://www.docker.com/products/docker-desktop/ 에서 .dmg 다운로드
#        → 더블클릭 → Docker.app 을 Applications 폴더로 드래그

open -a Docker              # Docker Desktop 실행
# 메뉴바 🐳 고래가 움직임 멈출 때까지 30~60초 대기

docker info >/dev/null 2>&1 && echo "✓ daemon up" || echo "✗ daemon down"
docker --version
docker compose version
```

## 7. 4단계 · 올라마 설치 (도서관 건물)

네 번째 단계가 올라마예요. 2장에서 *내 맥북 안에 사는 작은 동네 도서관* 으로 비유했죠. 인터넷 없이도 책 빌려 볼 수 있는 곳.

올라마 자체는 그렇게 크지 않아요. 한 100MB 정도. 금방 깔립니다. 그런데 **올라마 도서관 자체가 깔리는 것** 과 **그 도서관 안에 책(AI 모델) 이 들어 있는 것** 은 다른 얘기예요. 도서관 건물만 먼저 짓는 겁니다. 책장은 텅 비어 있어요. 책은 [6장](../part3-use/06-models.md) 에서 직접 골라서 채워 넣습니다.

이 단계가 끝났을 때 *"어, 책장이 비었는데요"* 하셔도 그게 정상이에요. 일단 도서관 건물부터 세운다.

> 💻 **직접 해보기** — 4단계 수동 버전: Ollama 설치 확인 (책장은 텅 비어 있음)

```bash
# (사전) https://ollama.com/download → .dmg → Applications 로 드래그 → 실행

ollama --version
ollama list                                       # 처음엔 비어 있음 (책장 텅)
curl -s http://localhost:11434/api/version        # {"version":"0.x.x"}
```

여기까지 따라오시면서 어떠세요. 머리 속이 좀 복잡해지셨죠. 괜찮습니다. 외우려고 하지 마세요. *"아, 한 줄짜리 명령어가 안에서는 이런 일들을 차례차례 하는구나"* 그 흐름만 느끼시면 됩니다.

## 8. 5단계 · 오픈클로 조립도 다운로드

다섯 번째 단계는 오픈클로 본체를 가져오는 거예요. 지금까지 우리가 깐 건 다 부품입니다. Xcode, Homebrew, 도커, 올라마. 이건 다 다른 회사가 만든 도구들이에요. 이걸 다 모아서 *"자, 이렇게 같이 쓰는 거예요"* 하고 조립도를 들고 있는 친구가 오픈클로입니다.

조립도를 인터넷에서 받아옵니다. 어려운 말로 **git clone** 인데, *"조립도 다운로드"* 정도로 생각하시면 돼요. 이 조립도가 우리 맥의 어느 폴더에 자리를 잡습니다. 보통 홈 폴더 아래 어디쯤이에요. 화면에 그 경로가 출력되는데, 한 번 눈여겨 봐주세요. 나중에 *"오픈클로 명령어 어디서 쳐요?"* 했을 때 가야 할 곳이 거기거든요. **이 폴더가 우리의 작업장**이에요. 보통 `openclaw-workspace` 라는 이름의 폴더입니다.

> 💻 **직접 해보기** — 5단계 수동 버전: 조립도(워크스페이스) clone

```bash
mkdir -p ~/DEV
cd ~/DEV
git clone https://github.com/GoGoComputer/openclaw-workspace.git
cd openclaw-workspace
ls                                                # README.md, openclaw-mgr/, docs/ 보이면 OK
```

## 9. 6단계 · 환경 설정 파일 (.env)

여섯 번째 단계가 좀 재밌어요. **환경 설정 파일** 을 만들어요. *점 이엔브이* 파일이라고 부르는데, 점이 앞에 붙어 있어요. 영어로 **dotenv**.

비유하자면 우리 집 부엌 서랍 속 비밀번호장 같은 거예요. 현관 비밀번호, 와이파이 비밀번호, 가스 검침원 올 시간. 가족들이 필요할 때 꺼내봐요. 우리 자동화 환경에도 그런 노트가 하나 필요합니다. 어떤 포트 번호로 통신할 거냐, 어떤 모델 이름을 기본으로 쓸 거냐, 이런 것들을 적어두는 노트예요.

인스톨러가 이걸 자동으로 만들어줍니다. 그리고 **이미 만들어 둔 게 있으면 덮어쓰지 않아요.** *"어, 이미 있네. 그럼 이건 그대로 두고, 새로 추가된 항목만 살짝 끼워 넣을게."* 이게 또 멱등성입니다. 두 번 돌려도 손으로 적어둔 비밀번호가 사라지지 않아요.

`.env.example` 의 전문(全文) 과 키별 의미는 [부록 B1](../appendix-b-env/B1-env-vars.md) 에 있습니다.

> 💻 **직접 해보기** — 6단계 수동 버전: `.env` 파일 준비

```bash
cd openclaw-mgr
cp .env.example .env
chmod 600 .env
# (필요 시) 편집:  open -e .env   또는  nano .env
# 최소 확인:       OPENCLAW_REPO, OPENCLAW_DIR
```

## 10. 7단계 · 컨테이너 깨우기

일곱 번째 단계, 이제 진짜로 컨테이너들을 깨웁니다. 지금까지는 부품을 모은 거였어요. 이제 이걸 가지고 **진짜로 작은 방들을 만들어요.** *"이쪽 방에는 웹 자동화 친구가 살고, 저쪽 방에는 보안 검사 친구가 살고, 또 다른 방에는 네트워크 도우미가 살고."*

도커는 부탁받은 대로 방들을 만듭니다. 한 방마다 1~2분 걸려요. 화면에 영어로 막 뭐가 지나갈 거예요. *"Pulling image"*, *"Creating container"*, *"Started"*. 한국말로 풀면 *"그림 받는 중"*, *"방 만드는 중"*, *"시작했어요"*. 영어가 무섭게 보이지만 안에서 일어나는 일은 단순합니다. 방을 만들고, 그 방에 친구를 들여보내고, 깨우는 거예요.

이때 합치는 yml 파일들의 전문(全文) 은 [부록 A7](../appendix-a-scripts/A7-compose.md) 에 있습니다.

> 💻 **직접 해보기** — 7단계 수동 버전: OpenClaw 본체 clone + 컨테이너 기동

```bash
# 7-a) OpenClaw 본체(공식 저장소) clone
OPENCLAW_DIR="${HOME}/openclaw"
OPENCLAW_REPO="$(grep '^OPENCLAW_REPO=' .env | cut -d= -f2-)"
[ -z "$OPENCLAW_REPO" ] && OPENCLAW_REPO="https://github.com/openclaw/openclaw.git"
git clone --depth 1 "$OPENCLAW_REPO" "$OPENCLAW_DIR"

# 7-b) 본체 .env 머지 (사용자 키 보존, 새 키만 추가)
SRC="$OPENCLAW_DIR/.env.example"; DST="$OPENCLAW_DIR/.env"
[ -f "$DST" ] || cp "$SRC" "$DST"
while IFS= read -r line; do
  case "$line" in ''|'#'*) continue;; esac
  key="${line%%=*}"
  grep -qE "^${key}=" "$DST" 2>/dev/null || echo "$line" >> "$DST"
done < "$SRC"
chmod 600 "$DST"

# 7-c) 컨테이너 기동 (보안 오버레이 포함)
cd "$OPENCLAW_DIR"
COMPOSE_FILES="-f docker-compose.yml"
[ -f compose.yml ] && COMPOSE_FILES="-f compose.yml"
SEC="$HOME/DEV/openclaw-workspace/openclaw-mgr/compose.security.yml"
[ -f "$SEC" ] && COMPOSE_FILES="$COMPOSE_FILES -f $SEC"
docker compose $COMPOSE_FILES up -d
docker compose $COMPOSE_FILES ps    # State=running 확인

# 7-d) 네트워크 격리 적용 (기본 isolated)
NET="$HOME/DEV/openclaw-workspace/openclaw-mgr/compose.network.yml"
[ -f "$NET" ] && COMPOSE_FILES="$COMPOSE_FILES -f $NET"
docker compose $COMPOSE_FILES up -d
mkdir -p "$HOME/.openclaw-mgr" && echo isolated > "$HOME/.openclaw-mgr/network-mode"
```

## 11. 8단계 · 최종 건강검진

여덟 번째가 마지막이에요. **건강검진**. 인스톨러가 마지막에 우리 맥을 쭉 둘러봅니다. *"도커 잘 돌아가나? 올라마 살아있나? 컨테이너들 다 깨어났나? 통신 잘 되나?"* 한 가지씩 체크해서, 다 초록불이면 *"축하합니다, 설치가 완료되었습니다"* 라고 뜹니다.

만약 중간에 어떤 친구가 빨간불이거나 노란불이면 *"이 친구가 좀 이상해요"* 하고 알려줘요. 그러면 그 친구만 다시 깨워주면 됩니다. 다시 깨우는 방법은 [4장](04-commands.md) 에서 배웁니다.

`doctor` 의 전문(全文) 은 [부록 A2](../appendix-a-scripts/A2-cmd-doctor.md), 빨간불이 떴을 때 따라가는 표는 [부록 C](../appendix-c-trouble/C1-by-symptom.md) 에 있습니다.

> 💻 **직접 해보기** — 8단계 수동 버전: doctor + HTTP 헬스체크

```bash
# 8-a) 워크스페이스 doctor — 깐 것 전체 진단
cd ~/DEV/openclaw-workspace/openclaw-mgr
./openclaw doctor                          # 모두 ✓ 면 OK

# 8-b) gateway HTTP 헬스 — 컨테이너가 응답하는지
curl -sS --max-time 5 -o /dev/null -w "HTTP %{http_code}\n" http://localhost:8000
# HTTP 200 = UI 준비 완료 → 브라우저: http://localhost:8000
```

## 12. 여덟 단계 요약

여기까지가 그 한 줄, `./openclaw install` 이 안에서 하는 일 여덟 단계예요.

> **첫째**, 자기소개와 맥 신분증 확인. **둘째**, 맥 기본 도구함 점검. **셋째**, 도커 데스크탑 설치. **넷째**, 올라마 설치. **다섯째**, 오픈클로 조립도 받기. **여섯째**, 비밀번호장 만들기. **일곱째**, 컨테이너 방 만들고 깨우기. **여덟째**, 건강검진.

이 여덟 단계를 한 줄에 묶어 놓은 게 그 명령어예요. 그러니까 우리가 그 한 줄을 칠 때, 우리는 사실 **여덟 가지 부탁을 한꺼번에 하고 있는 거**예요. 한 줄이 가벼워 보이지만, 누군가가 정성 들여 정리해 둔 여덟 단계의 응축입니다.

## 13. 직접 한 줄 쳐보기

자, 그러면 진짜로 한번 칩시다.

> 💻 **직접 해보기** — 처음부터 한 줄로 (조립도 다운로드 + 설치까지 죽 이어집니다)

```bash
# 부트스트랩 한 줄 — 폴더도 없고 명령어도 없을 때 첫 줄
git clone https://github.com/GoGoComputer/openclaw-workspace.git ~/openclaw-workspace \
  && cd ~/openclaw-workspace \
  && ./openclaw install
```

폴더가 이미 있는 분은 짧은 버전으로:

> 💻 **직접 해보기** — 위 1~8단계를 한 줄로 묶어 실행 (멱등 · 안전)

```bash
cd ~/DEV/openclaw-workspace/openclaw-mgr
./openclaw install
```

이 한 줄이 약간 닭이 먼저냐 달걀이 먼저냐 같은 문제를 풉니다. *"폴더가 아직 없는데 그 안의 명령어를 어떻게 치냐"* 라는. 부트스트랩 한 줄이 *"야 너 일단 폴더부터 가져와서, 그다음에 인스톨까지 다 좀 해줘"* 라고 부탁하는 거예요. 한 줄이 또 한 줄을 부르는 거죠.

## 14. 설치 중 보이는 강 · 색깔의 뜻

엔터를 치면 화면에 글자가 막 흘러갑니다. **다 읽으려고 하지 마세요.** 다 못 읽어요. 빠르거든요. 그냥 강물 보듯이 보세요. 색깔만 봐주세요.

| 색깔 | 뜻 |
|---|---|
| 🟢 **초록** | 잘 됐어요 |
| 🔵 **파랑** | 지금 진행 중 |
| 🟡 **노랑** | 음, 좀 봐줘야 할 수도 있는데 일단 넘어갈게요 |
| 🔴 **빨강** | 여기서 멈춰야 해요 |

빨간색이 나오면 그때만 좀 긴장하시면 됩니다. 그런데 빨간색이 나와도 너무 무서워하지 마세요. 우리는 어차피 멱등성이라는 안전장치가 있으니까, 다시 돌리면 되거든요.

## 15. 도커 데스크탑 팝업 대응법

도커 데스크탑이 깔리고 나면 화면에 *"Docker Desktop을 처음 실행하시려면 약관에 동의해 주세요"* 라는 팝업이 뜰 수 있어요. 비개발자분들이 여기서 가장 많이 당황하세요. 갑자기 영어 팝업이 떠서.

안 무서워요. **동의 누르세요.** *"통계 정보 보낼래요?"* 라고 물어보면 *"안 보낼래요"* 누르셔도 됩니다. 보낸다고 큰일 안 나요.

도커 데스크탑이 처음 켜지면 가입 화면이 뜰 수도 있어요. **가입 안 하셔도 됩니다.** 작은 글씨로 *"Skip"* 이라고 어딘가에 있을 거예요. 우리는 도커를 그냥 백그라운드 일꾼으로만 쓸 거예요. 도커 회사 이용자가 될 필요는 없습니다.

## 16. 설치 완료 — 새 동료 맞이하기

한 십 분 정도 흘러가다 보면 어느 순간에 글자 흐름이 멈춥니다. 마지막 줄에 이런 비슷한 메시지가 뜰 거예요.

```text
Installation complete.
```

또는 한국어로 *"설치가 완료되었습니다"*. 이게 뜨면 미션 완수입니다. **박수 치세요.** 진짜로요. 여러분 맥에 방금 자동화 도우미가 새로 입주했어요. 새 식구가 생긴 거예요.

이게 별거 아닌 것 같죠. 별거 맞아요. **십 년 전에 일주일이 걸리던 일을 십 분 만에 해내신 거예요.** 시간을 압축해서 거의 천 배쯤 빠르게 일을 하신 겁니다.

그런데 인생이 늘 그렇잖아요. 잘 안 되는 분도 계실 거예요. 그게 정상입니다. 이전에 깔아둔 다른 프로그램들, 인터넷 환경, 맥북 모델이 다 다르거든요. 빨간색이 뜨면 [부록 C · 트러블슈팅](../appendix-c-trouble/C1-by-symptom.md) 으로 가세요.

## 17. 직접 거울 보기 — `./openclaw doctor`

설치가 끝났으면 한 가지를 더 해보면 좋겠어요. 우리 자동화 도우미가 **정말로 잘 깔렸는지 직접 확인**하는 거예요. 인스톨러가 이미 건강검진을 해줬지만, 우리도 한번 해보자고요. 의사 선생님이 *"건강하세요"* 라고 해줘도, 우리가 직접 거울 한 번 보면 더 안심되잖아요.

그 거울을 보는 명령어가 `./openclaw doctor` 입니다. 닥터. 의사. 이름이 너무 직관적이죠. 닥터를 부르면 우리 자동화 환경 전체를 진찰해 줍니다. *"도커 살아있고, 올라마 살아있고, 컨테이너 다 깨어 있고, 통신 잘 되고, 디스크 공간 충분하고."*

> 💻 **직접 해보기** — 설치 끝난 직후, 독립적으로 건강검진 한 번 더

```bash
cd ~/openclaw-workspace
./openclaw doctor
```

다 초록색이면 안심하고 다음 장으로 넘어갈 수 있어요. 닥터가 *"이 친구가 좀 이상해요"* 하면 그 친구만 살짝 손봐주면 됩니다.

자, 마지막으로 한 가지만 더. 지금 이 설치 완료 메시지가 뜬 순간이 **여러분이 새 동료를 한 명 채용한 순간**이에요. 도크에 있는 그 고래 아이콘이 그 친구의 현관이고, 터미널이 그 친구랑 대화하는 인터폰이에요. 마음속으로 한 번 인사해 주세요. *"안녕, 잘 부탁해. 우리 잘 지내자."*

다음 장에서는 이 친구한테 일 시키는 법, **명령어 카탈로그**를 배웁니다.

## 🔬 깊이 보기

> *비개발자 독자는 이 박스를 건너뛰셔도 3장의 핵심을 다 얻으십니다.*

- 이 한 장에서 다룬 한 줄은 워크스페이스 루트의 `./openclaw install` 디스패처가 [`openclaw-mgr/cmd/install.sh`](../appendix-a-scripts/A1-cmd-install.md) 를 호출하는 흐름입니다.
- `install.sh` 는 대략 다음 순서로 동작합니다: (1) [`lib/detect.sh`](../appendix-a-scripts/A6-lib-rest.md) 로 OS·아키텍처 감지, (2) Xcode CLT/Homebrew 보장, (3) Docker Desktop 설치 및 시작 대기, (4) Ollama 설치, (5) 저장소 클론(이미 있으면 pull), (6) `.env` 머지(기존 키 보존), (7) [`compose.*.yml`](../appendix-a-scripts/A7-compose.md) 기반 컨테이너 기동, (8) 헬스체크.
- 각 단계는 [`lib/common.sh`](../appendix-a-scripts/A5-lib-common.md) 의 `run_step` 래퍼를 통해 상태 파일에 기록됩니다. 같은 단계가 이미 성공으로 마킹되어 있으면 스킵되며, 이것이 멱등성의 구현 단위입니다.
- 중단 후 재실행 시 [`~/.openclaw/state` 디렉터리](../appendix-b-env/B2-paths.md) 의 단계별 마커를 보고 이어서 실행합니다. 강제로 처음부터 돌리려면 해당 상태 파일을 지우거나 `./openclaw install --force` 옵션(존재한다면) 을 사용합니다.
- 빨간 에러 시 1차 진단은 `./openclaw doctor`, 2차는 `./openclaw logs <서비스명>`, 3차는 `docker compose ps` 로 컨테이너 상태 확인.
- `.env` 머지 로직은 사용자 지정 키를 덮어쓰지 않도록 작성되어 있으므로, 사용자 비밀값은 그대로 보존됩니다. 새 키만 추가됩니다.
