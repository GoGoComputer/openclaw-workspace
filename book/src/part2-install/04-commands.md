## 1. 사무실 풍경 — 새 동료 다섯 명

3장이 끝나고 도커 데스크탑을 한 번 열어보셨나요. 그 고래 화면이요. 안에 보면 컨테이너라는 탭이 있고, 거기에 작은 상자들이 몇 개 떠 있을 거예요. **그게 3장 마지막에 우리가 깨운 친구들**이에요. 다섯 마리쯤 보입니다. 어떤 친구는 초록불이고, 어떤 친구는 회색이에요.

- 🟢 **초록불** — 지금 일하고 있어요
- ⚪ **회색** — 지금은 자고 있어요, 부르면 일어나요

우리 새 동료는 다섯 명이 한 팀으로 살고 있어요. 이 장에서 배울 건 그 다섯 명한테 어떻게 말을 거는지, 어떻게 일을 시키는지, 어떻게 재우고 깨우는지, 그 **말 거는 법**입니다. 한국어로 명령어 카탈로그, 영어로 커맨드 카탈로그. 책 한 권의 목차 같은 거예요.

## 2. 마음의 준비 — 이미 아는 단어들

명령어라고 하면 보통 비개발자분들이 좀 긴장하세요. *"어, 무슨 코드 같은 거 외워야 하나"* 이 마음이 들어요. 안 그래도 됩니다. 이 장에서 배울 명령어는 다 영어 단어 한두 개예요.

- 시작 = **start** ("스타벅스" 의 그 스타)
- 멈춤 = **stop** ("스톱워치" 의 그 스톱)
- 일지 = **logs**
- 의사 = **doctor**

여러분 영어 못 하셔도 다 아시는 단어예요. 외울 게 한 열 개 정도 되는데, **그 열 개가 다 여러분 인생에서 한 번씩은 들어보신 영어 단어**들이에요. *"외워야 한다"* 가 아니라 *"이미 알고 있던 걸 어디에 어떻게 쓰는지만 배운다"* 라고 생각하시면 마음이 편해져요.

## 3. 한 곳에서만 친다 — 워크스페이스 폴더

이 명령어들은 **한 곳에서만 칠 거**예요. 작업장 폴더, `openclaw-workspace` 폴더. 거기 가서 거기서만 명령을 칩니다. 다른 데서 치면 우리 새 동료가 못 들어요. 그 친구들이 그 작업장에 살고 있거든요.

명령어를 칠 때마다 첫 동작은 항상 똑같습니다. *"내가 지금 작업장 안에 있나?"* 이걸 한 번 확인하는 거예요.

> 💻 **직접 해보기** — 작업장 폴더로 이동 + 위치 확인

```bash
cd ~/openclaw-workspace        # 또는 ~/DEV/openclaw-workspace 등 설치한 경로
pwd                            # 끝이 /openclaw-workspace 면 OK
ls                             # README.md, openclaw, openclaw-mgr/, scripts/ 보이면 OK
```

화면 위쪽에 폴더 이름이 보일 거예요. `openclaw-workspace` 라고 적혀 있으면 잘 들어와 있는 거예요. 우리는 여기저기 돌아다니지 말고, 일단 작업장에 들어가서, 거기서만 일을 합시다. 약속.

## 4. 명령어의 모양 — `./openclaw <동사>`

모든 명령어는 같은 모양입니다. **`./openclaw`** + 한 칸 + **동사 한 개**.

```text
./openclaw <동사>
```

점 슬래시는 *"이 폴더 안에 있는"* 이라는 뜻이고, openclaw 는 우리 동료의 이름이에요. *"야, 이 폴더 안에 있는 오픈클로야, 이 일 좀 해줘"* 라는 부탁을 매번 하는 거예요. 부탁의 내용이 뒤에 붙는 동사입니다.

디스패처의 전문(全文) 은 [부록 A11](../appendix-a-scripts/A11-dispatcher.md) 에 있습니다.

## 5. start — 다섯 동료 깨우기

첫 번째 동사, **스타트**.

> 💻 **직접 해보기**

```bash
./openclaw start
```

이건 우리 다섯 동료를 다 깨우는 명령이에요. 자고 있던 친구들을 다 일으켜서 일할 준비를 시킵니다. 보통 아침에 한 번 칩니다. 출근해서 컴퓨터를 켰을 때, *"자, 오늘도 일 좀 도와다오"* 하고 부르는 거예요. 다섯 친구가 차례로 일어나고, 옆에 초록색 점이 켜져요. 다 켜지는 데 한 10~30초 정도 걸립니다.

비유하자면 **사무실 문을 열고 들어가서 형광등 다섯 개를 차례로 켜는 거**예요. 한 번에 다 켜지지 않고 차례로 켜집니다.

그런데 잠깐, 3장 끝나고 우리 동료들은 이미 깨어 있을 거예요. 인스톨러가 마지막에 다 깨워주고 끝냈거든요. **그 상태에서 또 스타트를 치면?** 답은, 아무 일도 안 일어나요. 정확히는, 친구들이 *"어, 나 이미 깨어 있는데"* 하고 그냥 가만히 있어요. 이게 또 **멱등성**입니다. [7장](../part4-deep/07-internals.md) 에서 본격적으로 다루는데, 같은 명령을 두 번 쳐도 사고가 안 나요.

`start.sh` 의 전문(全文) 은 [부록 A3-1](../appendix-a-scripts/A3-cmd-lifecycle.md#a3-1-cmdstartsh) 에 있습니다.

## 6. stop — 잠시 재우기

두 번째 동사, **스톱**.

> 💻 **직접 해보기**

```bash
./openclaw stop
```

이건 반대예요. 다섯 친구를 다 재워요. 형광등 다섯 개를 차례로 꺼요. 사무실 문 닫고 나오는 거예요. 보통 노트북을 좀 가볍게 쓰고 싶을 때 칩니다. 컨테이너 다섯 개가 깨어 있으면 메모리도 좀 먹고, 가끔 팬이 돌고 그래요.

자고 일어나는 거 자체는 부담이 없어요. 사람이랑 다르게, 이 친구들은 자고 일어나도 피곤해하지 않아요. **마음 편하게 켜고 끄세요.**

`stop.sh` 의 전문(全文) 은 [부록 A3-2](../appendix-a-scripts/A3-cmd-lifecycle.md#a3-2-cmdstopsh) 에 있습니다.

## 7. down — 좀 더 강한 정리

스톱이랑 비슷한데 좀 강한 게 있어요. **다운**, 그러니까 *내려놓는다* 는 뜻. 친구들을 재우는 게 아니라 아예 방을 정리해서 보내버려요. 다음에 다시 부르려면 방을 새로 만들어야 해요.

이건 자주 쓰진 않아요. 보통은 스톱이면 충분합니다. 가끔 친구들이 좀 이상하게 행동할 때, 한 번 다 정리하고 새로 시작하고 싶을 때 다운을 씁니다.

비개발자분들은 일단 *"스톱이 있고, 더 강한 다운이 있다, 다운은 잘 안 쓴다"* 정도만 알고 계세요.

## 8. logs — 일지 보기

세 번째 동사, **로그스**. 이 장에서 진짜 자주 쓰실 겁니다.

> 💻 **직접 해보기**

```bash
./openclaw logs               # 다섯 친구 일지 전체
./openclaw logs surf          # surf 친구의 일지만
./openclaw logs surf -f       # -f = follow, 실시간 (빠져나오는 키 Ctrl+C)
```

로그가 뭐냐면, 우리 동료들이 *"내가 오늘 뭐 했어요"* 하고 적어두는 **일지**예요. 회사에서 인수인계할 때 쓰는 업무일지랑 똑같아요. 몇 시 몇 분에 누구 부탁을 받아서 무슨 일을 하다가 뭐가 잘 됐고 뭐가 안 됐는지 다 적혀 있어요.

*"어, 이 친구가 오늘 좀 이상한데"* 싶을 때 로그를 보면 거의 항상 답이 나옵니다. *"아, 어제 밤에 인터넷이 끊겨서 무슨 작업이 실패했었구나"* 라든지. *"아, 이 파일을 못 찾고 있구나"* 라든지. **거의 모든 문제는 로그에 답이 있어요.**

비개발자분들도 이 로그라는 단어 하나만은 꼭 친해지셨으면 좋겠어요. 무서워하지 마세요. 어차피 영어로 적혀 있어서 다 못 읽어요. 우리는 그냥 **빨간 줄이랑 ERROR 라고 적힌 줄만 찾으면 돼요.**

로그를 부르는 두 가지 방법이 있어요.

1. **전체** — `./openclaw logs` 만 치면 다섯 친구의 일지를 다 한꺼번에 보여줍니다. 정신없어요.
2. **한 친구만** — `./openclaw logs surf` 처럼 친구 이름을 같이 부릅니다. 한 사람 거니까 읽기가 훨씬 편해요.

친구 이름은 다섯 개예요. `surf`, `security`, `network`, `ollama`, 그리고 `gateway`. 일단 **`surf` 라는 이름은 외워두세요.** [5장](../part3-use/05-surf.md) 에서 진짜로 일 시킬 친구거든요.

끝에 `-f` 를 붙이면 (`./openclaw logs surf -f`) **실시간 따라가기 모드** 가 됩니다. F 는 follow 의 약자. 친구가 새로 적는 걸 어깨 너머로 같이 봐주는 거예요. 끝낼 때는 **Ctrl + C**. 외워두세요. 어디든지 **Ctrl + C 는 "여기서 나갈래요" 신호**입니다.

`logs.sh` 의 전문(全文) 은 [부록 A4-4](../appendix-a-scripts/A4-cmd-data.md#a4-4-cmdlogssh) 에 있습니다.

## 9. doctor — 건강검진

네 번째 동사, **닥터**.

> 💻 **직접 해보기**

```bash
./openclaw doctor
```

3장 끝에서 살짝 만났죠. 의사. 우리 동료들 건강검진. *"어, 뭐가 좀 이상한데, 도대체 어디가 이상한 거지?"* 할 때 칩니다. 닥터가 한 번 쭉 둘러보고 항목별로 초록불 빨간불을 보여줘요.

| 점검 항목 | 묻는 것 |
|---|---|
| 도커 데몬 | 살아있나 |
| 올라마 | 응답하나 |
| 디스크 | 공간 충분한가 |
| 인터넷 | 되나 |
| 컨테이너 | 다 깨어 있나 |
| 포트 | 안 막혔나 |

동네 의원 가는 거랑 똑같아요. 어디가 아픈지 모를 때 일단 가서 *"선생님, 어디가 안 좋은지 좀 봐주세요"* 하잖아요.

`doctor.sh` 의 전문(全文) 은 [부록 A2](../appendix-a-scripts/A2-cmd-doctor.md) 에 있습니다.

## 10. update · self-update

다섯 번째와 여섯 번째 동사 — **업데이트** 와 **셀프 업데이트**.

> 💻 **직접 해보기**

```bash
./openclaw update         # 동료(컨테이너) 이미지 새 버전
./openclaw self-update    # 오픈클로 매니저 자체를 새 버전
```

**업데이트** 는 우리 동료들을 새 버전으로 갈아입히는 거예요. 옷 갈아입히는 거. 시간이 지나면 동료들도 새 옷이 나와요. 더 빠르고, 더 안전하고, 새 기능도 들어 있는 옷이요. 이건 자주 안 쳐도 돼요. 한 달에 한 번이나 두 달에 한 번 정도. 옷 갈아입히는 동안 동료들은 잠깐 일을 못 해요. 그래서 **업무 시간 중에는 잘 안 칩니다.** 점심시간이나 퇴근 무렵에 쳐요.

**셀프 업데이트** 는 누가 옷을 갈아입느냐가 다릅니다. 그냥 업데이트는 다섯 동료가 갈아입어요. 셀프 업데이트는 **오픈클로 자기 자신**이 갈아입어요. 매니저 본인이 자기 자신을 새 버전으로 바꾸는 거예요. 한 분기에 한 번 정도. 일단 *"이런 게 있다"* 정도만 알고 계시면 됩니다.

[부록 A3-3 update](../appendix-a-scripts/A3-cmd-lifecycle.md#a3-3-cmdupdatesh) · [A3-6 self-update](../appendix-a-scripts/A3-cmd-lifecycle.md#a3-6-cmdself-updatesh).

## 11. backup · restore

일곱 번째와 여덟 번째 동사 — **백업** 과 **복원**.

> 💻 **직접 해보기**

```bash
./openclaw backup                  # 현재 자산 압축
./openclaw restore <백업파일경로>    # 압축 파일 풀기
```

**백업** 은 우리 동료들이 그동안 모아둔 것들, 만들어 둔 것들을 한 곳에 압축해서 보관하는 거예요. 사진 백업 하시잖아요. 그거랑 똑같은 개념이에요. 우리가 내려받은 AI 모델들, 만들어 둔 자동화 결과물들, 설정값들. 이게 다 우리 자산이에요. 한 번씩 꽁꽁 묶어서 따로 저장해 두면, 나중에 노트북이 망가지거나 새 노트북으로 옮길 때 그대로 살릴 수 있어요.

**복원** 은 백업의 반대예요. 어디 둔 그 백업 파일을 가져와서 풉니다. 이사 간 새집에 짐 풀듯이. 새 노트북에 오픈클로 인스톨 한 번 하시고 복원을 부르시면, **옛날 노트북에서 쓰던 환경이 그대로 살아나요.** 우리 자산이 노트북에 묶여 있지 않고 우리한테 묶여 있는 거예요.

백업은 [6장](../part3-use/06-models.md) 에서 본격적으로 다룹니다. [부록 A4-1](../appendix-a-scripts/A4-cmd-data.md#a4-1-cmdbackupsh) · [A4-2](../appendix-a-scripts/A4-cmd-data.md#a4-2-cmdrestoresh).

## 12. models · network

아홉 번째와 열 번째 동사 — **모델즈** 와 **네트워크**.

> 💻 **직접 해보기**

```bash
./openclaw models list             # 현재 가져다 둔 AI 모델
./openclaw models pull llama3.2    # 모델 다운로드
./openclaw models rm   llama3.2    # 모델 삭제
./openclaw network                 # 산책 코스(방화벽) 설정
```

**모델즈** 는 우리 도서관, 그러니까 올라마 도서관에 책을 채우는 명령어예요. 어떤 AI 모델을 다운로드 받을지, 지울지, 지금 어떤 모델들이 있는지 다 이걸로 관리합니다. [6장](../part3-use/06-models.md) 에서 같이 다룹니다.

**네트워크** 는 우리 동료들이 인터넷 어디로 나갈 수 있고 어디로는 못 나가게 하는, 그 통신 규칙을 정하는 거예요. 좀 어려운 단어로는 **방화벽 설정**. 비유로는 우리 동료들이 산책 나갈 수 있는 길을 정해주는 거예요. *"공원까지는 가도 돼, 옆 동네는 가지 마"* 이런 식.

자동화 도우미가 인터넷의 어디든 다 갈 수 있으면 좀 위험할 수 있어요. 우리가 모르는 곳에 우리 정보를 전송하거나, 이상한 곳에서 뭔가 받아오거나. 그래서 산책 코스를 정해주는 거예요. [7장](../part4-deep/07-internals.md) 에서 자세히 풀어드립니다.

[부록 A4-3 models](../appendix-a-scripts/A4-cmd-data.md#a4-3-cmdmodelssh) · [A4-6 network](../appendix-a-scripts/A4-cmd-data.md#a4-6-cmdnetworksh).

## 13. 열 개 동사 한 번에 외치기

자, 이렇게 열 개의 동사를 쭉 봤어요. 다시 한 번 외쳐 봅시다.

| 동사 | 한국어 | 일상에서도 쓰는 곳 |
|---|---|---|
| `start` | 시작 | 운전 |
| `stop` | 멈춤 | 스톱워치 |
| `logs` | 일지 | 회사 업무일지 |
| `doctor` | 진찰 | 병원 |
| `update` | 업데이트 | 핸드폰 앱 |
| `self-update` | 자기 갱신 | 자동 업데이트 |
| `backup` | 백업 | 사진 백업 |
| `restore` | 복원 | 옛날 사진 복원 |
| `models` | 모델 | 자동차 모델 |
| `network` | 네트워크 | 와이파이 |

별로 안 무섭죠. 다 일상에서 쓰는 단어예요. 컴퓨터 용어가 아니라 그냥 **생활 단어**. 우리는 새로 배운 게 아니에요. 알고 있던 걸 어디에 어떻게 쓰는지만 새로 배운 거예요.

## 14. 같이 쳐보기 — 라이브 시연

> 💻 **직접 해보기** — 동료들이 잘 자고 잘 일어나는지 확인

```bash
cd ~/openclaw-workspace
./openclaw doctor              # 첫 진찰
./openclaw logs                # 일지 전체 — Ctrl+C로 빠져나오기
./openclaw logs surf           # surf 친구만
./openclaw stop                # 다 재움
./openclaw doctor              # 재진찰 — 빨간불 안 나오는 게 정상
./openclaw start               # 다시 깨우기
./openclaw doctor              # 다 초록불이면 OK
```

지금 터미널을 한 번 열어주세요. 작업장 폴더로 들어가셨으면 **닥터**부터.

`./openclaw doctor` 엔터 — 한 줄씩 항목이 뜨면서 옆에 초록색 OK 또는 체크 표시가 뜹니다. 다 초록색이면 우리 환경이 건강한 거예요.

다음 **로그 전체**. `./openclaw logs` — 글자가 막 흘러갑니다. 다섯 친구 일지가 한꺼번에 흘러가요. 정신없죠. 이래서 보통 한 친구씩 봅니다.

**Ctrl + C** 로 빠져나오시고, **surf 친구만**. `./openclaw logs surf` — 좀 차분해졌죠. 영어가 많이 보이지만 빨간 줄만 보세요. 빨간 줄 없으면 surf 친구는 건강한 거예요.

이번엔 **재워봅시다**. `./openclaw stop` — 친구 이름이 하나씩 뜨면서 옆에 *"Stopped"* 라고 적힙니다. 다섯 친구 다 자고 있어요. 사무실 형광등 다 꺼졌어요.

그 상태에서 **닥터를 한 번 더**. `./openclaw doctor` — 이번엔 항목 몇 개에 빨간색이 보일 거예요. *"어, 이 친구 안 깨어 있어요"* 라고 닥터가 알려주는 거예요. 자고 있으니까요. **닥터는 거짓말 안 합니다.**

이제 **다시 깨워봅시다**. `./openclaw start` — 친구들이 하나씩 일어나요. 다 일어났으면 닥터 한 번 더. `./openclaw doctor` — 다시 다 초록색.

방금 우리가 한 게 뭐냐면, **새 동료한테 일을 시켜본 거**예요. *"건강 어때?"* 묻고, *"일지 좀 보여줘"* 묻고, *"다 자라"* 시키고, 다시 *"다 일어나"* 시키고. 진짜 동료처럼 대화를 한 거예요.

여기까지 따라오시면서 한 가지 깨달으시면 좋겠어요. **우리가 평생 쓸 명령은 한 열 개에서 스무 개 사이예요.** 영어 단어 스무 개. 자동화를 쓰는 데 필요한 어휘량은 사실 중학교 영어 단어장 한 페이지보다 적어요. 한 페이지요.

## 15. 외울 필요 없다 — `./openclaw help`

팁 하나 드릴게요.

> 💻 **직접 해보기**

```bash
./openclaw help
```

**헬프**. *도와줘*. 그러면 우리 매니저가 *"내가 알아듣는 말이 이런 게 있어요"* 하고 목록을 쭉 보여줍니다. **외울 필요가 없어요. 까먹으면 헬프 한 번 부르면 돼요.** 이게 또 도구의 힘이에요. 우리는 까먹어도 됩니다. 도구가 기억하고 있어요.

## 16. 옵션이란 무엇인가 — `-f`, `--help`, `--force`

명령어가 길어지는 경우도 있어요. `./openclaw logs surf -f` 처럼. 뒤에 붙는 게 **옵션**이에요. *"이 명령을 이렇게 좀 변형해서 해줘"* 라는 추가 부탁.

친구한테 커피 사다 달라고 할 때 *"아메리카노"* 라고만 하면 보통 아메리카노가 오죠. *"아이스로"* 라고 덧붙이면 아이스 아메리카노가 와요. 그 *"아이스로"* 가 옵션입니다.

옵션은 보통 **대시 한 개 또는 두 개로 시작**해요.

| 옵션 | 의미 |
|---|---|
| `-f` | follow (실시간 따라가기) |
| `--help` | 도움말 보기 |
| `--force` | 강제로 실행하기 |

자주 쓰는 옵션 두세 개만 알아두시면 한참 갑니다.

## 17. 오타 나도 괜찮아요

마지막으로 마음의 얘기 하나만요. 비개발자분들이 명령어를 처음 치실 때 가장 많이 하시는 실수가, 영어를 한 글자라도 잘못 칠까 봐 너무 긴장해서 손이 굳어요.

안 그러셔도 됩니다. **잘못 치면 어떻게 되냐.** 우리 매니저가 *"어, 내가 그런 말은 못 알아들어"* 하고 알려줘요. 그게 끝이에요. **큰일 안 나요. 아무것도 안 망가져요.** 그냥 한 줄 더 출력되고 끝이에요. 그러면 우리는 *"아 오타 났구나"* 하고 다시 치면 돼요.

이게 우리 동료들이 가진 또 하나의 미덕이에요. **우리가 잘못 말해도 화내지 않아요.** 그냥 *"그건 못 알아들어요"* 라고 차분하게 알려줘요. 손 떨지 마시고, 마음 편하게, 오타 좀 나도 괜찮다는 마음으로 쳐주세요.

## 18. 마무리

이 장에서 배운 게 **열 개의 동사** + **하나의 마법의 단어 (`help`)** 입니다.

다음 장에서는 이 동사들 중에서 가장 재밌는 친구, **`surf`** 라는 친구를 가지고 진짜로 웹 자동화를 한번 해볼 거예요. 진짜로 우리 동료가 우리 대신 인터넷을 둘러보고, 우리한테 정보를 가져다주는 걸 직접 보실 거예요. [1장](../part1-why/01-orientation.md) 에서 약속드린 그 환율 가져오기 같은 거.

> *비개발자 독자는 이 박스를 건너뛰셔도 4장의 핵심을 다 얻으십니다.*

- 모든 명령은 워크스페이스 루트의 `./openclaw` 디스패처를 통해 [`openclaw-mgr/cmd/<verb>.sh`](../appendix-a-scripts/A0-overview.md) 로 위임됩니다. 새 동사 추가는 `cmd/` 아래에 스크립트 파일 하나 추가하고 디스패처가 자동으로 인식하는 구조.
- `start`/`stop`은 내부적으로 `docker compose up -d` / `docker compose stop`에 매핑되며, 실제 컴포즈 파일은 [`compose.network.yml`, `compose.security.yml`, `compose.surf.yml`, `compose.ollama.yml`](../appendix-a-scripts/A7-compose.md) 등이 합쳐져 사용됩니다.
- `logs <service>`는 `docker compose logs <service>`에 매핑되고, `-f`는 그대로 follow 모드로 전달됩니다.
- `doctor`는 [`cmd/doctor.sh`](../appendix-a-scripts/A2-cmd-doctor.md) 로, 도커 데몬 상태, 컴포즈 서비스 상태, 포트 충돌, 디스크/메모리 여유, Ollama 응답 여부 등을 항목별로 점검합니다. 종료 코드를 보고 CI에서도 활용 가능.
- `update`는 컴포즈 이미지 pull 및 재기동을, `self-update`는 워크스페이스 저장소 자체의 git pull 및 디스패처 갱신을 수행합니다.
- `help`는 `cmd/` 폴더를 스캔해 사용 가능한 동사 목록을 출력하므로, 동사를 추가하면 별도 등록 없이 즉시 노출됩니다.
