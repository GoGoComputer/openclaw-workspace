## 1. 한 장 전과 지금

[5장](05-surf.md) 에서 진짜 자동화를 한 번 해보셨어요. 환율 가져오고, 캡처하고. **한 장 전에 여러분은 자동화를 한 번도 안 해본 사람이었고, 지금은 자동화를 해본 사람이에요.**

이 장은 그 자동화를 좀 더 오래, 안전하게 쓸 수 있게 해주는 **살림 얘기**예요. 백업, 복원, 그리고 우리 도서관에 책 채우기.

## 2. 백업이란 뭔가

백업이라는 단어, 우리 다 알아요. 사진 백업, 카톡 대화 백업, 핸드폰 백업. 일상에서 한 번씩은 들어본 단어예요. 그런데 **사람들이 백업을 진짜 안 해요.** 왜? 귀찮아서. 그리고 *"지금 당장 안 해도 안 죽으니까"*.

백업이라는 게 늘 그래요. 안 해도 오늘은 안 죽어요. 일주일 동안 안 해도 안 죽어요. 그러다가 어느 날 노트북이 망가져요. 또는 실수로 폴더를 지워요. 그 순간 깨달아요. *"아, 백업해 둘 걸."* 너무 늦었어요.

**백업은 망하기 전에 해두는 보험**이에요. 평소엔 돈만 나가는 것 같지만, 한 번 사고 나면 우리를 살려주잖아요.

## 3. 뭘 백업해야 하나 — 세 가지

| 분류 | 무엇 | 잃으면 |
|---|---|---|
| **환경 설정** | `.env`, `compose.*.yml` 등 | 동료들이 자기 일을 못 함 |
| **AI 모델** | 1GB ~ 40GB짜리 책들 | 다시 다운로드 (몇 시간) |
| **결과물** | 환율 마크다운, 캡처 이미지 등 | 돈으로 못 사는 데이터 |

세 번째가 특히 중요해요. **자동화로 매일 한 줄씩 쌓이는 결과물은 한 달이면 한 표가 되고, 한 해면 우리만의 데이터가 됩니다.** 잃으면 돈으로 못 사요.

## 4. 직접 해보기 — `./openclaw backup`

> 💻 **직접 해보기** — 첫 백업

```bash
cd ~/openclaw-workspace
./openclaw backup
```

화면에 뭐가 좀 흘러요.

```text
환경 설정 압축 중.
모델 압축 중.
결과물 압축 중.
백업 완료. 다음 위치에 저장했습니다: /Users/.../openclaw-backup/openclaw-2025-12-15.tar.gz
```

보통 사용자 홈 폴더 아래에 압축 파일 한 개로 저장됩니다.

이걸 어디에 두느냐가 중요해요. **같은 노트북에 두면 의미가 없어요.** 노트북이 망가지면 백업도 같이 망가지니까요.

| 추천 보관 위치 | 이유 |
|---|---|
| **iCloud Drive 폴더** | 가장 쉬움. 자동으로 클라우드에 올라감 |
| 외장하드 | 큰 백업도 보관 가능 |
| 회사 파일 서버 | 업무 환경이라면 |

`backup.sh` 의 전문(全文) 은 [부록 A4-1](../appendix-a-scripts/A4-cmd-data.md#a4-1-cmdbackupsh) 에 있습니다.

## 5. 백업 주기

| 시점 | 빈도 |
|---|---|
| 큰 변화 직후 (모델 추가, 설정 변경) | 그때그때 |
| 정기 백업 | 한 달에 한 번 |
| 휴가 가기 전 | 한 번 더 |

**매일 할 필요는 없어요.** 우리 자동화 환경은 매일 크게 바뀌지 않거든요. 한 달에 한 번이면 충분합니다.

## 6. 복원 — `./openclaw restore`

> 💻 **직접 해보기** — 새 노트북에서 옛 환경 살리기

```bash
./openclaw install                              # 새 노트북에서 먼저 환경 설치
./openclaw restore ~/Downloads/openclaw-2025-12-15.tar.gz   # 백업 파일 경로
```

복원은 백업의 반대예요. 백업 파일을 가져와서 새 환경에 풉니다. 보통 새 노트북에서 쓰는 명령이에요. 일단 [3장](../part2-install/03-install.md) 에서 했던 그 한 줄, `./openclaw install` 로 환경 깔고요, 그다음에 `./openclaw restore` 백업 파일 경로.

**옛날 환경이 새 노트북 안에 그대로 살아납니다.** 모델도, 설정도, 결과물도 다요. 노트북은 새 거인데 안에 들어 있는 우리 자동화 환경은 옛날 거랑 똑같아요. 이사 갈 때 짐을 그대로 싸 들고 가서 새집에서 옛 방 그대로 다시 꾸미는 거랑 비슷해요. **책상 위치까지 똑같이요.**

`restore.sh` 의 전문(全文) 은 [부록 A4-2](../appendix-a-scripts/A4-cmd-data.md#a4-2-cmdrestoresh) 에 있습니다.

## 7. 복원의 진짜 가치 — 노트북에서 자유롭게

복원의 진짜 가치는 **우리가 노트북에 묶이지 않게 해준다는 것**이에요.

자동화 환경이 노트북 안에만 살아있으면 그 노트북이 우리한테 족쇄가 돼요. 노트북 못 바꿔요. 회사 노트북에서만 자동화가 돌아가니까. 그런데 백업이랑 복원이 있으면 **자동화 환경이 우리한테 묶이는 거**예요. 가는 곳마다 따라옵니다.

> 회사 노트북에서도, 집 맥북에서도, 친구 컴퓨터에서도, 동일한 우리 환경이 돌아가요. 이게 디지털 노마드의 진짜 모습이에요. **도구가 우리를 따라다니는 것.**

## 8. 도서관 열기 — `./openclaw models`

> 💻 **직접 해보기** — 도서관 명령 카탈로그

```bash
./openclaw models list                # 현재 책장 장서 목록
./openclaw models pull <모델>         # 책 한 권 받아오기 (다운로드)
./openclaw models rm   <모델>         # 안 쓰는 책 빼기
```

[2장](../part1-why/02-concepts.md) 에서 올라마를 동네 도서관이라고 했죠. [3장](../part2-install/03-install.md) 에서 도서관 건물만 지었어요. 책장은 비어 있었어요. **이제 그 책장에 책을 꽂을 시간**입니다.

`models.sh` 의 전문(全文) 은 [부록 A4-3](../appendix-a-scripts/A4-cmd-data.md#a4-3-cmdmodelssh) 에 있습니다.

## 9. 추천 책 네 권

AI 모델은 한 종류만 있는 게 아니에요. 진짜 많아요. 수백 개 수천 개. 영어 잘하는 모델, 한국어 잘하는 모델, 코드 잘 짜는 모델, 그림 잘 보는 모델, 빠른 모델, 똑똑한 모델, 작은 모델, 큰 모델.

| 추천 | 모델 예시 | 크기 | 특징 | 좋은 일 |
|---|---|---|---|---|
| **1. 작고 빠른 한국어** | `llama3.2:3b` | ~4GB | 즉시 응답 / 깊은 생각은 좀 어설픔 | 메일 분류, 짧은 요약 |
| **2. 큰 생각쟁이** | `llama3.1:8b` | ~7GB | 다단계 작업 / 10~30초 걸림 | 긴 글 요약, 중요한 일 |
| **3. 한국어 품질 대안** | `qwen2.5:7b` | ~7GB | 한국어 품질 우수 | 한국어 위주 작업 |
| **4. 비전 (그림 보는)** | `llava:7b` | 7~40GB | 이미지 같이 봄 | 캡처 분석, 사진 설명 |

> 💻 **직접 해보기** — 추천 모델 다운로드

```bash
./openclaw models pull llama3.2:3b      # 작고 빠른 한국어 세팅 모델
./openclaw models pull llama3.1:8b      # 좀 더 큰 생각쟁이
./openclaw models pull qwen2.5:7b       # 한국어 품질 좋은 대안
```

화면에 다운로드 진행 막대기가 뜹니다. 4GB 면 5~10분 정도. 인터넷 속도에 따라요.

**두 친구를 받아두면, 일에 따라 둘을 골라 쓸 수 있어요.** 빠른 게 필요하면 작은 모델, 깊게 생각해야 하면 큰 모델.

비전 모델은 좀 큰 편입니다. 7GB ~ 40GB까지. **노트북 디스크가 충분한 분만** 받으세요.

## 10. 책장 정리 — list · rm

모델을 너무 많이 받으면 디스크가 꽉 차요. 100GB, 200GB 금방 갑니다. **안 쓰는 모델은 가끔 정리해 줘야 해요.**

> 💻 **직접 해보기** — 도서관 살림

```bash
./openclaw models list                  # 현재 책장 보기
./openclaw models rm llama3.2:3b        # 안 쓰는 책 빼기
```

`rm` 은 영어로 remove, *지운다* 는 뜻. 책장에서 책 한 권 빼서 버리는 거예요. **다시 필요하면 pull 로 또 받으면 돼요.**

`./openclaw models list` 한 번 같이 쳐봅시다. 책장에 꽂힌 책 목록이 쭉 떠요. 이름, 크기, 받은 날짜. 이게 우리 도서관의 장서 목록이에요. 가끔 한 번씩 들여다보세요. *"어, 이 책 안 본 지 오래됐네, 빼야겠다"* 하고요. 동네 도서관 사서들도 그렇게 정리하시거든요.

## 11. 왜 로컬 모델을 굳이 두는가

*"챗GPT나 클로드 같은 거 쓰면 더 좋은 모델 쓸 수 있잖아요. 왜 굳이 노트북에?"*

맞아요. 더 똑똑해요. 그런데 두 가지 단점이 있어요.

| 외부 모델의 단점 | 의미 |
|---|---|
| **데이터가 그 회사로 갑니다** | 회사 기밀, 개인 정보 들어 있는 일은 신경 쓰임 |
| **인터넷이 끊기면 못 써요** | 비행기, 약한 와이파이, 정전 시 사용 불가 |

그래서 노트북 안에 작은 도서관을 두는 거예요.

> **로컬 모델의 가치**
> - 인터넷 없어도 돌아감
> - 우리 데이터가 우리 노트북 밖으로 안 나감
>
> **자동화의 또 다른 철학** — 우리 도구는 우리 손에. 우리 데이터는 우리 노트북에.

## 12. 로컬과 외부 — 같이 쓰기

그렇다고 외부 모델을 안 쓰는 건 아니에요. **둘을 같이 씁니다.**

| 어떤 일 | 어떤 모델 |
|---|---|
| 평소 일상적인 일 | 우리 도서관 모델 (로컬) |
| 진짜 어려운 일 / 큰 모델 필요 | 외부 모델 (API) |

처음에는 그냥 우리 도서관 모델로만 시작하시는 게 마음 편해요. 외부 모델은 나중에 일이 익숙해지면 그때 추가하시면 됩니다.

## 13. 한 달에 한 번, 십 분 정리 습관

살림 잘하는 사람들의 공통점은 **정리하는 시간을 따로 둔다는 것**이에요.

| 시점 | 할 일 | 시간 |
|---|---|---|
| 매달 첫 토요일 오전 | 백업 1회 + 모델 목록 점검 + 안 쓰는 모델 1~2개 빼기 | 10분 |

**십 분짜리 정리 시간이 우리 자동화 환경을 1년, 10년 건강하게 유지해 줘요.** 이걸 안 하면 도서관이 잡동사니로 가득 차게 됩니다. 안 쓰는 모델 열 개에, 안 쓰는 결과물 폴더 백 개에, 디스크는 꽉 차고. 그 상태에서 자동화하기가 점점 부담스러워집니다.

작은 정리 습관을 처음부터 들이세요. **한 달에 한 번, 십 분.** 약속.

## 14. 마무리

이 장에서 우리는:

- **백업** 으로 우리 환경을 보험에 들었고
- **복원** 으로 노트북에서 자유로워지는 법을 배웠고
- **모델 pull** 로 도서관에 책 네 권 추천 목록을 받았고
- **모델 관리** 로 도서관 살림하는 법을 익혔어요

이제 우리 자동화 환경은 단단해요. 망가져도 살릴 수 있고, 옮겨도 따라오고, 책장은 우리가 골라서 채울 수 있고. **이 정도면 진짜 우리 거예요. 우리 자산이에요.**

다음 장은 그동안 가볍게 지나갔던 단어들 — **멱등성**, **격리**, **보안** — 그 원리들을 풀어드립니다. 이 원리만 알면 자동화를 더 안심하고 쓸 수 있어요.

## 🔬 깊이 보기

> *비개발자 독자는 이 박스를 건너뛰셔도 6장의 핵심을 다 얻으십니다.*

- `./openclaw backup` 은 [`cmd/backup.sh`](../appendix-a-scripts/A4-cmd-data.md#a4-1-cmdbackupsh) 를 실행, `.env`/컴포즈 파일/올라마 모델 디렉터리 (`~/.ollama/models` 등) /사용자 산출물 디렉터리를 `tar.gz` 로 묶어 사용자 지정 위치 (기본은 `~/openclaw-backup/`) 에 저장합니다.
- 모델 디렉터리는 가장 큰 비중을 차지하므로 백업에서 모델을 제외하는 옵션 (`--no-models`) 이 제공될 수 있습니다. 이 경우 복원 후 다시 `models pull` 이 필요합니다.
- `./openclaw restore <archive>` 는 압축 해제 후 `.env` 머지 (기존 값 보존), 모델 디렉터리 복원, 컴포즈 재기동을 수행합니다. **멱등하게 설계되어 같은 아카이브를 두 번 복원해도 안전**합니다.
- `./openclaw models` 는 내부적으로 `ollama` CLI 를 호출하는 래퍼이며, `pull`/`list`/`rm`/`run` 등의 서브커맨드를 그대로 노출합니다.
- 외부 API 모델 사용이 필요한 경우 `.env` 에 API 키를 추가하고 에이전트 라우팅 설정 (보통 [`scripts/*-lib/`](../appendix-a-scripts/A8-scripts-creative.md) 에 있는 모델 선택 로직) 에서 분기 처리합니다.
- 백업 파일은 민감 정보 (API 키 등이 포함된 `.env`) 를 담을 수 있으므로 외부 저장 시 **암호화 또는 접근 제어가 권장**됩니다.
