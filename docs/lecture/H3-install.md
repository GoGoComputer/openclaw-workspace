### 7단계 — 컨테이너 방 만들고 깨우기 (OpenClaw 본체 clone + compose up)

```bash
# 7-a) OpenClaw 본체(공식 저장소) clone
OPENCLAW_DIR="${HOME}/openclaw"
OPENCLAW_REPO="$(grep '^OPENCLAW_REPO=' .env | cut -d= -f2-)"
[ -z "$OPENCLAW_REPO" ] && OPENCLAW_REPO="https://github.com/openclaw/openclaw.git"
git clone --depth 1 "$OPENCLAW_REPO" "$OPENCLAW_DIR"

# 7-b) 본체 .env 머지 (사용자 키 보존, 새 키만 추가)
SRC="$OPENCLAW_DIR/.env.example"; DST="$OPENCLAW_DIR/.env"
[ -f "$DST" ] || cp "$SRC" "$DST"
while IFS= read -r line; do
  case "$line" in ''|'#'*) continue;; esac
  key="${line%%=*}"
  grep -qE "^${key}=" "$DST" 2>/dev/null || echo "$line" >> "$DST"
done < "$SRC"
chmod 600 "$DST"

# 7-c) 컨테이너 기동 (보안 오버레이 포함)
cd "$OPENCLAW_DIR"
COMPOSE_FILES="-f docker-compose.yml"
[ -f compose.yml ] && COMPOSE_FILES="-f compose.yml"
SEC="$HOME/DEV/openclaw-workspace/openclaw-mgr/compose.security.yml"
[ -f "$SEC" ] && COMPOSE_FILES="$COMPOSE_FILES -f $SEC"
docker compose $COMPOSE_FILES up -d
docker compose $COMPOSE_FILES ps    # State=running 확인

# 7-d) 네트워크 격리 적용 (기본 isolated)
NET="$HOME/DEV/openclaw-workspace/openclaw-mgr/compose.network.yml"
[ -f "$NET" ] && COMPOSE_FILES="$COMPOSE_FILES -f $NET"
docker compose $COMPOSE_FILES up -d
mkdir -p "$HOME/.openclaw-mgr" && echo isolated > "$HOME/.openclaw-mgr/network-mode"
```

### 🪟 Windows 학생용 명령 묶음 (강의 후 집에서)

```powershell

.\openclaw.ps1 schedule enable          # 매일 자동 update (작업 스케줄러 = launchctl 등가)
```

### 1. 옛날엔 일주일짜리, 지금은 한 줄

## 1. 들어가며 — 한 줄의 의미

자, 3교시입니다. 다들 잠깐 숨 한 번 쉬시고요. 1교시에서 우리가 왜 자동화를 배우는지 이야기했고, 2교시에서는 터미널, 도커, 올라마, AI 에이전트 이 네 단어를 정리했습니다. 이제 드디어, 진짜로, 우리 맥북 안에 이 친구들을 살게 만들 시간입니다. 제목이 뭐였죠. 설치 한 줄의 의미. 한 줄. 명령어 딱 한 줄이에요. 그 한 줄이 도대체 무슨 일을 하는지, 왜 그게 대단한 건지, 그리고 그 한 줄이 실패하면 우리가 뭘 어떻게 해야 하는지, 이 한 시간 동안 천천히 같이 따라가 봅시다.

## 2. 일주일이 걸리던 설치 — 양파 까기

먼저 옛날 얘기를 잠깐 할게요. 제가 처음 자동화라는 걸 회사에서 만져봤을 때가 한 십 년쯤 전이에요. 그때는 어떤 자동화 도구 하나를 내 컴퓨터에 깔려면, 진짜로 일주일이 걸렸어요. 일주일이요. 농담 아니에요. 왜냐하면 일단 설치 안내 문서가 있는데, 그 문서가 한 사십 페이지짜리예요. 그리고 첫 페이지부터 이렇게 시작합니다. "먼저 파이썬 3.7 이상이 설치되어 있어야 합니다." 그러면 저는 파이썬을 깔러 갑니다. 파이썬을 까는데 또 뭐가 필요해요. 뭔가 컴파일러가 필요하대요. 컴파일러를 깔려면 또 다른 게 필요하고요. 이게 끝없이 이어져요. 제가 그때 썼던 표현이 뭐였냐면, 양파 같다. 까도 까도 안에 또 다른 게 나와요. 그리고 그 양파를 다 까고 나면 이제 진짜 그 자동화 도구를 까는데, 이번엔 환경 변수라는 걸 설정해야 한대요. 그게 또 뭐예요. 그러면 또 검색합니다. 그렇게 일주일이 지나고 나면 어떤 일이 벌어지냐면, 제가 처음에 왜 이걸 깔려고 했는지를 까먹어요. 진짜예요. 분명히 어떤 업무 하나 자동화 하려고 시작한 건데, 일주일을 헤매다 보면 "내가 지금 뭐 하고 있지" 이 상태가 됩니다. 그리고 그 시점에서 한 절반 정도는 포기해요. "아, 됐다. 그냥 손으로 하자." 이게 자동화가 우리 일상으로 안 들어왔던 가장 큰 이유 중에 하나예요. 시작하기가 너무 어려워서.

## 3. 누군가가 깍아둔 양파 — `./openclaw install`

그런데 시간이 지나면서, 이 양파 까는 일을 누가 좀 대신해 주면 안 되나, 라는 생각을 사람들이 하기 시작합니다. 그래서 나온 게 우리가 이제 만질 그 한 줄짜리 명령어예요. `./openclaw install`. 점, 슬래시, 오픈클로, 스페이스, 인스톨. 끝이에요. 이 한 줄을 터미널에 입력하고 엔터를 치면, 방금 제가 십 년 전에 일주일 동안 했던 그 모든 양파 까기를, 이 친구가 알아서 합니다. 알아서. 한 십 분에서 이십 분 정도 걸려요. 그동안 여러분은 커피 한 잔 드시면 돼요. 진짜로요. 이게 왜 대단한 일인지, 비개발자분들은 처음엔 잘 와닿지 않으실 거예요. "원래 그런 거 아니에요?" 라고 생각하시기 쉬워요. 아니에요. 원래 그런 거 아니었어요. 누군가가 그 양파 까는 작업을 한 줄짜리 스크립트 안에 차곡차곡 정리해 둔 거예요. 그래서 우리가 그 한 줄만 치면 되는 거예요. 그러니까 우리가 지금부터 하는 일은, 누군가가 우리를 위해서 미리 깎아 둔 양파를 받아오는 거예요. 정말 감사한 일이죠.

## 4. 한 줄 안을 들여다보기 — 여덟 단계 개관

자, 그러면 이 한 줄이 실제로 안에서 뭘 하는지 한번 같이 들여다봅시다. 들여다본다고 해도 코드 보자는 얘기는 아니에요. 그냥 "어떤 일들이 순서대로 일어나는지" 그림으로 그려보자는 거예요. 머리 속에 이 그림이 한 번 그려지면, 나중에 어디서 멈춰도 "아 지금 어디쯤이구나" 감이 와요. 그게 오늘 3교시의 진짜 목표예요.

## 5. 1단계 · 손님맞이 준비 (맥 신분증 확인)

첫 번째로 하는 일이 뭐냐면, 손님맞이 준비예요. 자기소개를 합니다. "안녕하세요, 저는 오픈클로 인스톨러예요. 지금부터 이 맥북에 자동화 환경을 만들어 드릴게요." 이렇게 화면에 좀 친절하게 인사를 해요. 그리고 한 가지 중요한 걸 물어봅니다. "지금 이 맥북, 인텔이에요 애플 실리콘이에요?" 왜 묻냐면요, 인텔 맥이랑 M1, M2, M3, M4 같은 애플 실리콘 맥이랑은 안에서 돌아가는 부품이 좀 달라요. 그래서 깔아야 할 도커 버전도 다르고, 올라마가 쓸 수 있는 AI 모델 종류도 좀 달라져요. 그래서 인스톨러가 가장 먼저 우리 맥의 신분증을 한번 확인하는 거예요. 이게 무슨 어려운 일이 아니에요. 그냥 시스템한테 "너 누구니" 한번 물어보는 거예요. 그리고 그 답에 따라서 뒤에 깔 부품을 달리 준비하는 거죠. 자, 여기까지가 첫 번째 단계, 한 일 분 정도 걸려요.

> ▶ **같이 쳐보기** — 1단계 수동 버전: 맥 신분증만 확인

```bash
uname -m            # arm64 = Apple Silicon / x86_64 = Intel
sw_vers             # macOS 버전
```

## 6. 2단계 · 도구함 점검 (Xcode CLT + Homebrew)

두 번째 단계가 뭐냐면, 도구함 점검이에요. 맥에는 원래부터 들어 있는 도구들이 있어요. 그중에 우리가 꼭 필요한 게 두 개가 있는데, 하나는 엑스코드 커맨드 라인 툴이라는 거고, 다른 하나는 홈브루라는 거예요. 이름은 어렵죠. 안 외우셔도 됩니다. 풀어 말하면 이래요. 엑스코드 커맨드 라인 툴은 맥이 새로운 프로그램을 깔 때 쓰는 기본 연장 세트예요. 망치, 드라이버, 펜치 같은 거. 이게 없으면 다른 친구들이 자기 집을 못 지어요. 홈브루는 뭐냐면, 맥용 앱 스토어인데 터미널에서 쓰는 버전이에요. 우리가 평소에 앱스토어에서 카카오톡 깔고 슬랙 깔잖아요. 그거랑 똑같은데, 마우스로 클릭하는 게 아니라 터미널에서 글자로 "이거 깔아줘" 하면 알아서 깔아주는 친구예요. 인스톨러가 두 번째 단계에서 이 둘이 있는지 한 번 확인하고, 없으면 깔아줍니다. 있으면 그냥 "오, 이미 있네, 패스" 하고 넘어가요. 이게 좀 중요한 포인트예요. 이미 있으면 또 깔지 않아요. 이걸 어려운 말로 멱등성이라고 하는데, 7교시에서 본격적으로 다룰 거예요. 지금은 그냥 "있으면 안 깔고 넘어간다, 그래서 두 번 세 번 돌려도 안전하다" 정도로만 기억해 두시면 돼요.

> ▶ **같이 쳐보기** — 2단계 수동 버전: Xcode CLT 설치 + git 확인

```bash
xcode-select --install      # 다이얼로그 → "설치" 클릭 → 5~10분
git --version               # git version 2.x = OK
```


## 7. 3단계 · 도커 데스킱 설치

세 번째가 진짜 큰 친구입니다. 도커 데스크탑이에요. 2교시에서 우리가 도커를 뭐라고 했죠. 컨테이너라는 작은 방을 만드는 친구. 작업장 한쪽에 칸막이로 나눈 공방. 그 도커를 이 단계에서 깔아요. 도커 데스크탑은 용량이 좀 커요. 한 칠백 메가에서 일 기가 정도 되거든요. 그래서 인터넷 속도에 따라서 좀 시간이 걸려요. 화면에 막대기 같은 게 슥슥 차오르는 게 보이실 거예요. 그게 다운로드 받는 중이라는 표시예요. 다 받으면 그걸 또 깔아요. 깔고 나면 한 가지 더 해줍니다. 도커가 백그라운드에서 돌아가도록 시작 버튼을 눌러줘요. 이건 진짜로 우리가 아무것도 안 해도 되는 부분이에요. 그냥 보고만 계시면 돼요. 비개발자분들 중에 "어, 도커 데스크탑이라는 게 도크에 새로 생겼어요" 하시는 분 계실 거예요. 맞아요. 그거예요. 고래 모양 아이콘. 그 친구가 이제부터 우리 맥 한쪽에 살면서 컨테이너들을 관리해 줄 거예요. 도크에 있는 그 고래 보이시면, 마음속으로 한 번 인사해 주세요. "안녕, 잘 부탁해."

> ▶ **같이 쳐보기** — 3단계 수동 버전: Docker Desktop 실행 + 데몬 확인

```bash
open -a Docker              # Docker Desktop 실행
docker info >/dev/null 2>&1 && echo "✓ daemon up" || echo "✗ daemon down"
docker --version
docker compose version
```

## 8. 4단계 · 올라마 설치 (도서관 건물만 먼저)

네 번째 단계가 올라마예요. 이것도 2교시에서 얘기했죠. 우리 맥 안에 사는 작은 동네 도서관. 인터넷 없이도 책 빌려 볼 수 있는 곳. 이 올라마를 이 단계에서 깔아요. 올라마 자체는 그렇게 크지 않아요. 한 백 메가 정도. 금방 깔려요. 그런데 올라마라는 도서관 자체가 깔리는 거랑, 그 도서관 안에 책, 그러니까 AI 모델이 들어 있는 거랑은 다른 얘기예요. 도서관 건물만 먼저 짓는 거예요. 책장은 텅 비어 있어요. 책은 나중에, 6교시에서 우리가 직접 골라서 채워 넣을 거예요. 그러니까 이 단계가 끝났을 때 "어, 책장이 비었는데요" 하셔도 그게 정상이에요. 맞게 가고 있는 거예요. 일단 도서관 건물부터 세운다. 이게 네 번째 단계의 핵심이에요.

> ▶ **같이 쳐보기** — 4단계 수동 버전: Ollama 설치 확인 (책장은 텍 비어 있음)


자 잠깐. 여기까지 따라오시면서 어떠세요. 머리 속이 좀 복잡해지셨죠. 괜찮아요. 외우려고 하지 마세요. 지금 우리는 그냥 "아, 한 줄짜리 명령어가 안에서는 이런 일들을 차례차례 하는구나" 그 흐름만 느끼면 돼요. 흐름만요. 단계 이름 외우는 거 아니에요. 시험 안 봐요. 우리는 시험 보러 온 거 아니에요. 우리는 우리 일을 더 편하게 하러 온 거예요. 그러니까 마음 편하게, 그냥 강물 흘러가듯이 들어주세요.

## 9. 5단계 · 오픈클로 조립도 다운로드

다섯 번째 단계는 오픈클로 본체를 가져오는 거예요. 지금까지 우리가 깐 건 다 부품이에요. 엑스코드, 홈브루, 도커, 올라마. 이건 다 다른 회사가 만든 도구들이에요. 이걸 다 모아서 "자, 이렇게 같이 쓰는 거예요" 하고 조립도를 들고 있는 친구가 오픈클로예요. 그래서 다섯 번째 단계에서 오픈클로의 조립도를 인터넷에서 받아옵니다. 이걸 좀 어려운 말로 깃 클론이라고 하는데, 그냥 "조립도 다운로드" 정도로 생각하시면 돼요. 이 조립도가 우리 맥의 어느 폴더에 자리를 잡게 됩니다. 보통은 여러분 홈 폴더 아래 어디쯤에 만들어져요. 화면에 그 경로가 출력되는데, 그거 한 번 눈여겨 봐주세요. 나중에 우리가 "오픈클로 명령어 어디서 쳐요?" 했을 때 가야 할 곳이 거기거든요. 이 폴더가 우리의 작업장이에요. 우리가 앞으로 한 학기, 아니 한 평생 자동화 작업을 할 우리의 공방이에요. 이름 한 번 봐 두시면 좋아요. 보통 openclaw-workspace 라는 이름의 폴더예요.

> ▶ **같이 쳐보기** — 5단계 수동 버전: 조립도(워크스페이스) clone

```bash
mkdir -p ~/DEV
cd ~/DEV
git clone https://github.com/GoGoComputer/openclaw-workspace.git
cd openclaw-workspace
ls                                                # README.md, openclaw-mgr/, docs/ 보이면 OK
```

## 10. 6단계 · 환경 설정 파일 (.env)

여섯 번째 단계가 좀 재밌어요. 환경 설정 파일을 만들어요. 이걸 점 이엔브이 파일이라고 부르는데, 점이 앞에 붙어 있어요. 점 이엔브이. 영어로 닷 이엔브이. 이건 뭐냐면, 우리 자동화 환경이 쓸 비밀번호장 같은 거예요. 비유하자면 우리 집 현관 비밀번호, 와이파이 비밀번호, 가스 검침원 올 시간, 이런 걸 적어둔 작은 노트가 있다고 생각해 보세요. 그 노트를 부엌 서랍 안에 넣어두면, 가족들이 필요할 때 꺼내봐요. 우리 자동화 환경에도 그런 노트가 하나 필요해요. 어떤 포트 번호로 통신할 거냐, 어떤 모델 이름을 기본으로 쓸 거냐, 이런 것들을 적어두는 노트예요. 인스톨러가 이걸 자동으로 만들어 줍니다. 그리고 이미 만들어 둔 게 있으면 덮어쓰지 않아요. "어, 이미 있네. 그럼 이건 그대로 두고, 새로 추가된 항목만 살짝 끼워 넣을게." 이렇게 합니다. 이게 또 멱등성이에요. 두 번 돌려도 여러분이 손으로 적어둔 비밀번호가 사라지지 않아요. 안심하세요.

> ▶ **같이 쳐보기** — 6단계 수동 버전: `.env` 파일 준비

```bash
cd openclaw-mgr
cp .env.example .env
chmod 600 .env
# (필요 시) open -e .env  또는  nano .env
```

## 11. 7단계 · 컨테이너 깨우기

일곱 번째 단계, 이제 진짜로 컨테이너들을 깨웁니다. 지금까지는 부품을 모은 거였어요. 도커도 깔았고, 올라마도 깔았고, 조립도도 받았고, 환경 설정도 했어요. 이제 이걸 가지고 진짜로 작은 방들을 만들어요. "이쪽 방에는 웹 자동화 친구가 살고, 저쪽 방에는 보안 검사 친구가 살고, 또 다른 방에는 네트워크 도우미가 살고." 이렇게 도커한테 부탁을 합니다. 도커는 부탁받은 대로 방들을 만들어요. 한 방마다 한 일이 분 정도 걸려요. 화면에 영어로 막 뭐가 지나갈 거예요. "Pulling image", "Creating container", "Started". 이런 말들이 보일 텐데, 한국말로 풀면 "그림 받는 중", "방 만드는 중", "시작했어요" 이 정도예요. 영어가 무섭게 보이지만, 안에서 일어나는 일은 진짜 단순해요. 방을 만들고, 그 방에 친구를 들여보내고, 그 친구를 깨우는 거예요. 그게 다예요.

> ▶ **같이 쳐보기** — 7단계 수동 버전: OpenClaw 본체 clone + 컨테이너 기동

```bash
# 7-a) OpenClaw 본체 clone
OPENCLAW_DIR="${HOME}/openclaw"
git clone --depth 1 https://github.com/openclaw/openclaw.git "$OPENCLAW_DIR"

# 7-b) 컴포즈 기동 (보안 + 네트워크 오버레이 포함)
cd "$OPENCLAW_DIR"
MGR="$HOME/DEV/openclaw-workspace/openclaw-mgr"
docker compose -f docker-compose.yml \
  -f "$MGR/compose.security.yml" \
  -f "$MGR/compose.network.yml" up -d
docker compose ps    # State=running 확인
```

## 12. 8단계 · 최종 건강검진

여덟 번째 단계가 마지막이에요. 건강검진. 인스톨러가 마지막에 한번 우리 맥을 쭉 둘러봐요. "도커 잘 돌아가고 있나? 올라마 살아있나? 컨테이너들 다 깨어났나? 통신은 잘 되나?" 한 가지씩 체크해서, 다 초록불이 들어오면 "축하합니다, 설치가 완료되었습니다" 라고 뜹니다. 만약에 중간에 어떤 친구가 빨간불이거나 노란불이면, "이 친구가 좀 이상해요" 하고 알려줘요. 그러면 우리는 그 메시지를 보고, 그 친구만 다시 깨워 주면 됩니다. 다시 깨우는 방법은 4교시에서 배울 거예요. 일단 지금은 "아, 마지막에 건강검진을 해주는구나" 이것만 알고 계시면 돼요.

> ▶ **같이 쳐보기** — 8단계 수동 버전: doctor + HTTP 헬스철크

```bash
cd ~/DEV/openclaw-workspace/openclaw-mgr
./openclaw doctor                          # 모두 ✓ 면 OK
curl -sS --max-time 5 -o /dev/null -w "HTTP %{http_code}\n" http://localhost:8000
# HTTP 200 = UI 준비 완료 → 브라우저: http://localhost:8000
```

## 13. 여덟 단계 요약

자, 여기까지가 그 한 줄, `./openclaw install`이 안에서 하는 일 여덟 단계예요. 한번 머리 속에서 다시 그려볼게요. 첫째, 자기소개와 맥 신분증 확인. 둘째, 맥 기본 도구함 점검. 셋째, 도커 데스크탑 설치. 넷째, 올라마 설치. 다섯째, 오픈클로 조립도 받기. 여섯째, 비밀번호장 만들기. 일곱째, 컨테이너 방 만들고 깨우기. 여덟째, 건강검진. 이 여덟 단계를 한 줄에 묶어 놓은 게 그 명령어예요. 그러니까 우리가 그 한 줄을 칠 때, 우리는 사실 여덟 가지 부탁을 한꺼번에 하고 있는 거예요. "야 이거 다 좀 부탁해" 하고. 그리고 인스톨러는 그 부탁을 토 달지 않고 다 들어줘요. 이게 한 줄의 의미예요. 한 줄이 가벼워 보이지만, 실은 누군가가 정말 정성 들여 정리해 둔 여덟 단계의 응축이에요. 그래서 저는 이 한 줄을 칠 때마다 마음속으로 한 번 감사를 해요. 누군가가 이걸 미리 정리해 줬구나, 덕분에 내가 일주일 헤맬 일을 십 분 만에 끝내는구나, 라고요. 여러분도 한번 그렇게 마음을 가지고 엔터를 쳐보시면 어떨까 싶어요.


화면을 한번 봐주세요. 제가 지금 새 터미널을 하나 열게요. 까만 화면 하나가 떴죠. 여기에 글자를 칠 거예요. 첫 번째로, 우리가 이 한 줄을 칠 폴더로 가야 해요. 어디로 가야 한다고 했죠. 오픈클로 워크스페이스 폴더요. 그 폴더가 아직 없는 분도 계실 거예요. 처음 깔으시는 거니까. 그러면 그 폴더가 없는 상태에서 어떻게 그 안의 명령어를 치겠어요. 이게 약간 닭이 먼저냐 달걀이 먼저냐인데, 그래서 우리는 두 가지 방법 중 하나를 씁니다. 하나는 먼저 그 폴더를 깃이라는 친구한테 부탁해서 가져오는 거고, 다른 하나는 더 간단한 방법인데, 한 줄짜리 부트스트랩 스크립트를 받아서 도는 거예요. 한 줄짜리 부트스트랩이라는 건 "야 너 일단 폴더부터 가져와서, 그다음에 인스톨까지 다 좀 해줘" 라고 부탁하는 한 줄이에요. 그러니까 한 줄이 또 한 줄을 부르는 거죠. 인셉션 같죠. 화면에 보이는 이 한 줄을 그대로 복사해서 터미널에 붙여 넣으세요. 붙여 넣고 엔터 치시면 돼요.

## 15. 설치 중 보이는 강 · 색깔의 뜻

엔터를 치셨죠. 자, 화면에 글자가 막 흘러갈 거예요. 지금 흘러가는 글자를 다 읽으려고 하지 마세요. 다 못 읽어요. 빠르거든요. 그냥 강물 보듯이 보세요. 색깔만 좀 봐주세요. 초록색, 파란색, 노란색이 주로 보이실 거예요. 초록은 "잘 됐어요" 예요. 파란색은 "지금 진행 중이에요" 예요. 노란색은 "음, 이거 좀 봐줘야 할 수도 있는데, 일단 넘어갈게요" 예요. 빨간색이 나오면 그때만 좀 긴장하시면 돼요. 빨간색은 "여기서 멈춰야 해요" 거든요. 그런데 빨간색이 나와도 너무 무서워하지 마세요. 우리는 어차피 멱등성이라는 안전장치가 있으니까, 다시 돌리면 되거든요.

## 16. 도커 데스탅 팝업 대응법

아, 그리고 한 가지요. 도커 데스크탑이 깔리고 나면, 화면에 "Docker Desktop을 처음 실행하시려면 약관에 동의해 주세요" 라는 팝업이 뜰 수 있어요. 비개발자분들이 여기서 가장 많이 당황하세요. 갑자기 영어 팝업이 떠서. 안 무서워요. 동의 누르시고, 만약에 "통계 정보 보낼래요?" 라고 물어보면 "안 보낼래요" 누르셔도 돼요. 보낸다고 해도 큰일 안 나요. 그냥 도커 회사가 "사람들이 우리 제품 어떻게 쓰는지 좀 알고 싶어요" 하는 거예요. 그건 여러분 마음대로 하시면 됩니다. 그리고 도커 데스크탑이 처음 켜지면 가입 화면이 뜰 수도 있어요. 가입 안 하셔도 돼요. 작은 글씨로 "Skip" 이라고 어딘가에 있을 거예요. 그거 누르시면 됩니다. 가입을 굳이 하실 이유가 없어요. 우리는 도커를 그냥 백그라운드 일꾼으로만 쓸 거예요. 도커 회사 이용자가 될 필요는 없어요.

## 17. 설치 완료 메시지와 박수

자, 이렇게 한 십 분 정도 흘러가다 보면, 어느 순간에 글자 흐름이 멈춰요. 그리고 마지막 줄에 이런 비슷한 메시지가 뜰 거예요. "Installation complete." 또는 한국어로 "설치가 완료되었습니다." 이게 뜨면 우리는 1단계 미션 완수입니다. 자, 박수 한 번. 진짜로요. 박수 치세요. 왜냐하면 여러분 맥에 방금 자동화 도우미가 새로 입주했어요. 새 식구가 생긴 거예요. 이건 박수받을 일이에요. 비개발자분들, 진짜 잘하셨어요. 이게 별거 아닌 것 같죠. 별거 맞아요. 십 년 전에 일주일이 걸리던 일을 여러분은 십 분 만에 해내신 거예요. 시간을 압축해서 거의 천 배쯤 빠르게 일을 하신 거예요. 이게 도구의 힘이에요. 우리가 이 한 학기, 아니 이 하루 동안 배우는 게 바로 이거예요. 도구의 힘을 빌리는 법.

## 18. 안 되는 분을 위한 약속

그런데, 인생이 늘 그렇잖아요. 잘 안 되는 분도 계실 거예요. 그게 정상이에요. 제가 강의를 한 백 번쯤 했는데, 단 한 번도, 단 한 번도 모든 분이 한 번에 다 성공한 적이 없어요. 항상 한두 분은 어디서 막히세요. 그게 여러분 잘못이 아니에요. 그건 우리가 쓰는 맥북이 다 다르고, 인터넷 환경이 다 다르고, 이전에 깔아둔 다른 프로그램들이 다 달라서 그래요. 그래서 미리 약속 하나 할게요. 만약에 화면에 빨간색이 떠서 "어, 안 됐어요" 가 뜨면, 손을 드세요. 손을 드시고 "강사님 빨간색이요" 한 마디만 해주세요. 그러면 우리가 같이 봐드릴게요. 부끄러워하지 마세요. 옆 사람이랑 비교하지 마세요. 옆 사람도 자기 화면 보느라 정신없어요. 여러분만 봐드리는 시간이에요.

그리고 만약에 아예 이 한 줄을 치는 단계까지도 못 가신 분, 그러니까 터미널을 어떻게 여는지부터 헷갈리시는 분도 계실 거예요. 그것도 정상이에요. 1교시에서 우리가 약속했죠. 못 따라오시는 분이 한 분이라도 있으면 거기서 다시 시작한다고. 손 들어 주세요. 우리는 같이 가요. 빨리 가는 게 목적이 아니에요. 다 같이 도착하는 게 목적이에요.

## 20. 마무리 — 새 식구를 맞이하며

자 이제 마지막으로 한 가지만 더 얘기하고 3교시를 마무리할게요. 이게 좀 감성적인 얘기예요. 여러분, 지금 이 순간이 사실 특별한 순간이에요. 왜냐하면 여러분 맥북에 방금, 자동화 도우미가 입주했어요. 보이지는 않지만 거기 살고 있어요. 도크에 있는 그 고래 아이콘이 그 친구의 현관이고, 터미널이 그 친구랑 대화하는 인터폰이에요. 여러분이 자고 있을 때도 그 친구는 거기 있어요. 여러분이 회사에서 일할 때도, 카페에서 커피 마실 때도, 그 친구는 거기 살고 있어요. 그리고 여러분이 부르면 언제든지 일을 해줍니다. "야, 환율 좀 가져와 봐" 하면 가져오고, "메일 좀 정리해 봐" 하면 정리하고. 우리가 이제부터 한 시간씩 가르칠 거예요. 그 친구한테 일 시키는 법을. 그러니까 지금 이 설치 완료 메시지가 뜬 순간이, 여러분이 새 동료를 한 명 채용한 순간이에요. 마음속으로 한 번 인사해 주세요. "안녕, 잘 부탁해. 우리 잘 지내자." 이렇게요.

> 👨‍💻 개발자 노트 (참고 — 비개발자는 그냥 넘기셔도 됩니다)
>
> - 이 한 시간에서 다룬 한 줄은 저장소 루트의 `./openclaw install` 디스패처가 `openclaw-mgr/cmd/install.sh`를 호출하는 흐름입니다.
> - install.sh는 대략 다음 순서로 동작합니다: (1) `lib/detect.sh`로 OS·아키텍처 감지, (2) Xcode CLT/Homebrew 보장, (3) Docker Desktop 설치 및 시작 대기, (4) Ollama 설치, (5) 저장소 클론(이미 있으면 pull), (6) `.env` 머지(기존 키 보존), (7) `compose.*.yml` 기반 컨테이너 기동, (8) 헬스체크.
> - 각 단계는 `lib/common.sh`의 `run_step` 래퍼를 통해 상태 파일에 기록됩니다. 같은 단계가 이미 성공으로 마킹되어 있으면 스킵되며, 이것이 멱등성의 구현 단위입니다.
> - 중단 후 재실행 시 `~/.openclaw/state` 디렉토리(또는 그에 준하는 위치)의 단계별 마커를 보고 이어서 실행합니다. 강제로 처음부터 돌리려면 해당 상태 파일을 지우거나 `./openclaw install --force` 옵션(존재한다면)을 사용합니다.
> - 빨간 에러 시 1차 진단은 `./openclaw doctor`, 2차는 `./openclaw logs <서비스명>`, 3차는 `docker compose ps`로 컨테이너 상태 확인을 권장합니다.
> - `.env` 머지 로직은 사용자 지정 키를 덮어쓰지 않도록 작성되어 있으므로, 사용자 비밀값을 그대로 보존합니다. 새 키만 추가됩니다.
